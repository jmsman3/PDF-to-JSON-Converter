# Roadmap to Growth

This roadmap explains how to grow the repository gradually so it becomes more useful, more discoverable, and easier for people to trust and share.

## Phase 1: Strong first impression

- Keep the README clear and focused.
- Keep the repo public-safe and easy to run.
- Show one sample output file.
- Show one business use case.
- Put the main value in the top section.

## Phase 2: Useful features

- Add batch conversion for multiple PDFs.
- Add output folder support.
- Add page-range selection.
- Add OCR support notes for scanned PDFs.
- Improve JSON metadata for automation workflows.

## Phase 3: Contributor momentum

- Label starter issues clearly.
- Respond quickly to issues and pull requests.
- Keep the contributing guide simple.
- Add release notes for each version.

## Phase 4: Visibility

- Share the repo on LinkedIn, X, and relevant communities.
- Use real examples like sales lead PDFs or contact lists.
- Add screenshots or short demos.
- Ask users to star, fork, and give feedback.

## Phase 5: Ongoing activity

- Ship small updates regularly.
- Close stale issues politely.
- Improve docs when users get confused.
- Add new examples based on real feedback.

## Practical advice

The safest growth strategy is to make the repo genuinely helpful first, then promote it with real use cases.

That usually leads to better stars, forks, and contributor interest than forcing hype too early.
